                                                    # 简介

                                                    Markdown 是一种轻量级标记语言，由 John Gruberopen in new window 和 Aaron Swartzopen in new window 在 2004 年创建，具有轻量化、易读易写等特性。

                                                    # 标准化

                                                    Markdown 在创建之初只支持日常文档的格式排版，如标题、列表等。随着时间的推移，人们在此基础实现了一些额外的功能，如表格、脚注等。由于这些实现的语法没有一个明确的标准，这导致不同的 Markdown 应用程序在同一个实现上有自己的语法规则。

                                                    为了促使 Markdown 语法的标准化，从 2012 年开始，包括 Jeff Atwood 和 John MacFarlane 在内的一群人启动了标准化工作。但在 2014 年 9 月，John Gruber 反对这一工作继续使用 “Markdown” 这个名字，他认为不应该完全标准化：“Different sites (and people) have different needs. No one syntax would make all happy.”。最后这项工作被更名为 CommonMark。

                                                    CommonMark.orgopen in new window 发布了规范、参考实现和测试包的几个版本，并在 2019 年发布了最终的 1.0 规范和测试包。在 2017 年，Github 发布了基于 CommonMark 的 Github Flavored Markdown (GFM) 正式规范，扩展了对表格、删除线、自动链接和任务列表的实现。

                                                    # 语法分类

                                                    Markdown 的语法可以分为两类，分别是基础语法和扩展语法。

                                                    # 基础语法

                                                    基础语法是 John Gruber 创建 Markdown 时定义的语法，包含以下元素：

                                                    • 标题
                                                    • 段落与换行
                                                    • 强调
                                                    • 引用
                                                    • 列表
                                                    • 代码
                                                    • 链接
                                                    • 图片
                                                    • 分隔线
                                                    • 转义字符
                                                    • 内嵌 HTML 标签

                                                    # 扩展语法

                                                    扩展语法是一些个人或组织在基础语法上，添加了其他实现的语法。

                                                    提示

                                                    我的笔记都是放到 Github 上面进行版本管理的，所以这里介绍的的扩展语法是 GFM 所支持的。

                                                    一般包括以下元素：

                                                    • 表格
                                                    • 代码块
                                                    • 删除线
                                                    • 任务列表

                                                    块元素与行内元素

                                                    你可能会在后面的文章看到有些元素被称为块元素，而有些元素被称为行内元素。

                                                    如果你熟悉 HTML，这两个概念可能会让你感到困惑，所以在此简单说明 Markdown 中块元素和行内元素的定义与分类。

                                                    • 块元素：与其他元素之间需要使用一个或以上空白行分隔的元素
                                                    • 行内元素：作用于一行中某些文本的元素

                                                    空白行是指行内不包含任何内容或只有空白字符的行。

                                                    基础语法中，标题、段落、引用、列表、图片、分隔线都是块元素，换行、强调、代码和链接都是行内元素。

                                                    对于扩展语法，你可以根据上面的定义对元素进行分类，上面所提到的表格、代码块、删除线和任务列表都是块元素。

                                                    # 写在最后

                                                    一般来说，所有的 Markdown 应用程序都应该支持基础语法。

                                                    对于扩展语法，每个 Markdown 应用程序都有自己所支持的扩展语法，有些收费的 Markdown 编辑器会以独创的扩展语法作为卖点。

                                                    所以，当一个网站或应用程序说他们支持 Markdown 的时候，你需要阅读他们的文档，遵循他们所规定的语法。

                                                    Markdown 应用程序 和 Markdown 编辑器的区别

                                                    Markdown 应用程序指的是可以解析 Markdown 语法的程序，它可以作为一部分嵌入到网站或其他应用程序中。

                                                    Markdown 编辑器结合了文本编辑器和 Markdown 应用程序，是专门用来写 Markdown 文档的应用程序。

                                                    如果看到这里的你准备入门 Markdown，我有两个建议：

                                                    1. 刚开始不要使用 WYSIWYG 的编辑器，Windows 用户（包括 macOS ）推荐使用 Visual Studio Codeopen in new window，Linux 用户推荐使用 Vimopen in new window，这两个都是纯文本编辑器，可以通过配置看到空白符
                                                    2. 对于扩展语法，尽可能选择有广泛支持的，这可以确保你的 Markdown 文档在不同的平台有较好的移植性。GFM 是一个不错的选择。Github 作为全球最大的代码免费托管平台，使得 GFM 在各大平台，各类 Markdown 编辑器都有不错的支持，而且你可以使用 Github 来管理和保存 Markdown 文档，不用担心文档丢失的问题。
