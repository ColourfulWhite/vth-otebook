            # 简介

            Java 语言是一门高级语言，具有面向对象、强静态类型的特点。

            严格来说，Java 并不是纯编译型的语言，它需要经过先编译、后解释的步骤。

            Java 的源文件 *.java 通过 javac 编译成与平台无关的 *.class 字节码文件，然后由 Java 虚拟机 (JVM) 解释执行。得益于 JVM，Java 语言在跨平台方面有优秀的表现。

            # 名词解释

            • JVM

              JVM 全称 Java Virtual Machine，是 Java 语言能够跨平台的核心部分。所有平台的 JVM 向上给编译器提供统一的编程接口，向下适应不同平台的接口。编译器面向 JVM 生成 JVM 能够理解的 *.class 字节码文件，再由 JVM 进行解释运行。

            • JRE

              JRE 全称 Java Runtime Environment，也称为 Java 运行时环境。一个 Java 程序的执行，不仅需要 JVM 进行解释，还需要其他的类加载器、字节码校验器以及基础类库等。JRE 就是这些工具和资源的集合。

              如果只需要运行 Java 程序而不用开发，可以只安装 JRE。

            • JDK

              JDK 全称 Java SE Development Kit，也称 Java 标准版开发包，包含了编译、运行 Java 程序所需要的各种工具和资源，包括 Java 编译器、Java 运行时环境以及常用的 Java 类库。

            • Java SE

              Java SE (Standard Edition) 是标准版的 Java 平台，主要用于开发桌面应用、控制台应用和一般的 Java 应用程序。

              它包括 Java 语言的核心功能和标准的 API，用于处理基本的任务，如输入输出、数据结构、多线程等。

              Java SE 适用于一般的应用程序开发，不涉及特定的领域或应用服务器。

            • Java EE

              Java EE (Enterprise Edition) 是企业版的 Java 平台，专门设计用于开发大型、分布式、企业级应用程序。

              它构建在 Java SE 的基础上，提供了额外的 API 和功能，用于处理企业级应用的需求，如 Web 应用、分布式系统、事务管理、安全性和数据库访问。

              Java EE 通常与 Java 应用服务器（如 Tomcat 等）一起使用，以提供企业级应用程序的部署和管理。

            • Java ME

              Java ME (Micro Edition) 是微型版的 Java 平台，专门设计用于嵌入式和移动设备上的应用程序开发。

              它包括一个小型的 Java 运行时环境，适合资源受限的设备，如智能手机、嵌入式系统和物联网设备。

              Java ME 通常用于开发小型应用程序，如手机游戏、智能卡应用和传感器应用。
