                                # Nginx 版本

                                Nginx 官方网站下载页面：https://nginx.org/en/download.htmlopen in new window

                                Nginx 官方网站的下载页面提供了两种版本，分别是 Mainline version 和 Stable version。

                                • Mainline version：开发版本，包含了最新的一些功能，但不稳定
                                • Stable version：稳定版本

                                建议在生产环境使用 Stable version。

                                # 系统准备

                                # 内核

                                Nginx 使用 epoll 模型处理高并发的访问，确保系统的内核是 2.6 及以上版本。可以使用 uname -a 查看系统的内核版本。

                                $ uname -a
# 当前内核版本为 5.15.0，架构为 aarch64 
Linux raspberrypi 5.15.0-1044-raspi #47-Ubuntu SMP PREEMPT Tue Nov 21 11:33:46 UTC 2023 aarch64 aarch64 aarch64 GNU/Linux

                                # 防火墙

                                由于 Nginx 提供网络访问服务，所以需要在防火墙配置规则，对 Nginx 的流量进行放行。

                                不同的系统使用到防火墙不同，Ubuntu 22.04 使用到是 UFW 防火墙，可以通过 sudo ufw allow 'Nginx Full' 配置，允许HTTP（端口 80）和 HTTPS（端口 443）流量，如果使用了其他端口，需要手动把规则加入到防火墙中。

                                建议在学习的过程中，把防火墙停止运行。

                                Ubuntu
                                sudo systemctl stop ufw
sudo systemctl disable ufw
                                CentOS
                                sudo systemctl stop firewalld
sudo systemctl disable firewalld

                                # SELinux

                                SELinux（Security-Enhanced Linux）是一个 Linux 安全模块，它提供了强大的访问控制机制，用于加强 Linux 系统的安全性，在基于 Red Hat 的发行版系统，例如 Red Hat Enterprise Linux (RHEL)、CentOS 和 Fedora，一般会默认启动 SELinux。除了 SELinux，类似的安全模块还有 Apparmor， Ubuntu 就是使用 Apparmor 作为系统默认的安全模块。

                                这些安全模块虽然安全，但会让 Nginx 在使用过程中增加额外的配置，建议在学习 Nginx 时关掉。

                                Apparmor
                                sudo systemctl stop apparmor
sudo systemctl disable apparmor
                                SELinux

                                把 /etc/selinux/config 的 SELINUX 配置项设置成 disable，并重启系统即可。

                                sudo vim /etc/selinux/config
sudo reboot

                                # 使用包管理器安装

                                Nginx 官方提供了一些常见的 Linux 发行版本的预编译包，可以通过在包管理器添加 Nginx 的仓库，并通过包管理器对 Nginx 进行安装。

                                具体参考：https://nginx.org/en/linux_packages.htmlopen in new window

                                安装完成后， 使用 nginx 启动 Nginx，使用 whereis nginx 查看 Nginx 安装的位置。

                                # Ubuntu

                                1. 安装使用到的一些工具

                                  sudo apt install curl gnupg2 ca-certificates lsb-release ubuntu-keyring

                                2. 导入 Nginx 官方的签名信息

                                  curl https://nginx.org/keys/nginx_signing.key | gpg --dearmor \
    | sudo tee /usr/share/keyrings/nginx-archive-keyring.gpg >/dev/null

                                3. 验证签名信息

                                  gpg --dry-run --quiet --no-keyring --import --import-options import-show /usr/share/keyrings/nginx-archive-keyring.gpg

                                  确保输出如下所示，如果不是，重新进行第 2 步。

                                  pub   rsa2048 2011-08-19 [SC] [expires: 2024-06-14]
      573BFD6B3D8FBC641079A6ABABF5BD827BD9BF62
uid                      nginx signing key <signing-key@nginx.com>

                                4. 导入 Nginx 官方的 apt 源

                                  Stable
                                  echo "deb [signed-by=/usr/share/keyrings/nginx-archive-keyring.gpg] \
http://nginx.org/packages/ubuntu `lsb_release -cs` nginx" \
    | sudo tee /etc/apt/sources.list.d/nginx.list
                                  Mainline
                                  echo "deb [signed-by=/usr/share/keyrings/nginx-archive-keyring.gpg] \
http://nginx.org/packages/mainline/ubuntu `lsb_release -cs` nginx" \
    | sudo tee /etc/apt/sources.list.d/nginx.list

                                5. 设置优先使用官方的 apt 源而不是系统的 apt 源

                                  echo -e "Package: *\nPin: origin nginx.org\nPin: release o=nginx\nPin-Priority: 900\n" \
    | sudo tee /etc/apt/preferences.d/99nginx

                                6. 安装 Nginx

                                  sudo apt update
sudo apt install nginx

                                # CentOS

                                1. 安装使用到的工具

                                  sudo yum install yum-utils

                                2. 设置 yum 源

                                  在 /etc/yum.repos.d/ 下面新建一个 nginx.repo 文件，添加以下内容：

                                  [nginx-stable]
name=nginx stable repo
baseurl=http://nginx.org/packages/centos/$releasever/$basearch/
gpgcheck=1
enabled=1
gpgkey=https://nginx.org/keys/nginx_signing.key
module_hotfixes=true

[nginx-mainline]
name=nginx mainline repo
baseurl=http://nginx.org/packages/mainline/centos/$releasever/$basearch/
gpgcheck=1
enabled=0
gpgkey=https://nginx.org/keys/nginx_signing.key
module_hotfixes=true

                                3. 安装 Nginx

                                  默认使用的是 Stable 版本，如果需要使用 Mainline 版本，执行 sudo yum-config-manager --enable nginx-mainline。

                                  sudo yum install nginx

                                # 编译源码安装

                                # 环境准备

                                Nginx 的源码可以在官方的下载页面找到，但可能版本不全，可以在 https://nginx.org/download/open in new window 页面找到 Nginx 所有版本的源码，其中以 .zip 结尾的是适用于 Windows 系统的源码。

                                在编译 Nginx 的源码之前，需要准备编译的环境，确保系统安装了 GCC、PCRE、zlib、OpenSSL。

                                • GCC：Nginx 使用 C 语言编写，需要使用 GCC 对源码进行编译
                                • PCRE：PCRE(perl Compatible Regular Expression) 是兼容正则表达式库，为 Nginx 的 Rewrite 模块 和 HTTP 核心模块提供正则表达式的支持
                                • zlib：zlib 库为 Nginx 提供 gzip 等压缩算法的支持
                                • OpenSSL：OpenSSL 为 Nginx 实现 HTTPS 通信所需的加密和安全功能提供支持

                                安装这些软件可以使用系统自带的包管理器：

                                Ubuntu
                                sudo apt install build-essential libpcre3 libpcre3-dev zlib1g zlib1g-dev libssl-dev
                                CentOS
                                sudo yum install gcc pcre pcre-devel zlib zlib-devel openssl openssl-devel

                                安装完成后查看是否安装成功：

                                Ubuntu
                                dpkg-query -l gcc make libpcre3 libpcre3-dev zlib1g zlib1g-dev libssl-dev
                                CentOS
                                rpm -qa gcc make pcre pcre-devel zlib zlib-devel openssl openssl-devel

                                # 编译安装

                                官方文档页面：https://nginx.org/en/docs/configure.htmlopen in new window

                                下载 Nginx 的源码，并进行解压，并按以下步骤进行编译安装。

                                tar -zxf nginx-1.24.0.tar.gz

                                1. 使用 configure 配置 Makefile

                                  在解压后的目录中，使用 configure 程序来生成编译时用到的 Makefile 文件。执行 ./configure --help 可以查看可配置的编译参数。这些参数大致可以分为三类：

                                  • PATH 类：这类配置用于配置程序的一些路径
                                  • with 类：这类配置用于控制模块加入到编译 Nginx 的过程
                                  • without 类：这类配置用于控制模块不加入到编译 Nginx 的过程

                                  对于 PATH 类，常见的配置项作用如下：

                                  配置项描述默认值
                                  --prefix=PATHNginx 的安装目录/usr/local/nginx/
                                  --sbin-path=PATH可执行文件 nginx 的路径<prefix>/sbin/nginx
                                  --modules-path=PATHNginx 动态模块安装目录<prefix>/modules/
                                  --conf-path=PATH配置文件 nginx.conf 的路径<prefix>/conf/nginx.conf
                                  --error-log-path=PATH错误日志文件的路径<prefix>/logs/error.log
                                  --http-log-path=PATH访问日志文件的路径<prefix>/logs/access.log
                                  --pid-path=PATHNginx 启动后进程 ID 的文件路径<prefix>/logs/nginx.pid
                                  --lock-path=PATHNginx 锁文件的路径<prefix>/logs/nginx.lock
                                  ./configure --prefix=/usr/local/nginx \
--sbin-path=/usr/local/nginx/sbin/nginx \
--modules-path=/usr/local/nginx/modules \
--conf-path=/usr/local/nginx/conf/nginx.conf \
--error-log-path=/usr/local/nginx/logs/error.log \
--http-log-path=/usr/local/nginx/logs/access.log \
--pid-path=/usr/local/nginx/logs/nginx.pid \
--lock-path=/usr/local/nginx/logs/nginx.lock

                                2. 执行 make 进行编译

                                  make

                                3. 执行 make install 进行安装

                                  make install

                                4. 启动 Nginx

                                  cd /usr/local/nginx/sbin/
./nginx

                                5. 添加到环境变量

                                  通过编译安装的 Nginx，每次执行 Nginx 的命令时都需要进入到 Nginx 的可执行文件目录或者输入绝对路径，可以将这个目录加入到环境变量或者在 /usr/local/bin 中为 Nginx 添加一个符号链接。注意：需要根据实际安装路径填写 Nginx 的可执行文件所在的路径，也就是编译选项中 --sbin-path 的值！

                                  • 修改环境变量

                                    当前用户

                                    在 ~/.bashrc 最后加入 export PATH=$PATH:/usr/local/nginx/sbin，并执行 source ~/.bashrc 使其生效。

                                    echo 'export PATH=$PATH:/usr/local/nginx/sbin' >> ~/.bashrc

                                    source ~/.bashrc
                                    全部用户

                                    在 /etc/profile 最后加入 export PATH=$PATH:/usr/local/nginx/sbin，并执行 source /etc/profile 使其生效。

                                    echo 'export PATH=$PATH:/usr/local/nginx/sbin' >> /etc/profile

                                    source /etc/profile

                                  • 添加符号链接

                                    sudo ln -s /usr/local/nginx/sbin/nginx /usr/local/bin/nginx

                                # 卸载

                                卸载 Nginx 之前需要先停止，执行 nginx -s stop 停止运行 Nginx。

                                # 使用包管理器安装方式的卸载

                                Ubuntu

                                只卸载 Nginx：

                                sudo apt remove nginx

                                卸载并删除相关的配置文件：

                                sudo apt purge nginx
                                CentOS

                                只卸载 Nginx：

                                sudo yum remove nginx

                                卸载并删除相关的配置文件：

                                sudo yum erase nginx

                                # 编译安装方式的卸载

                                直接删除编译安装 Nginx 时 --prefix 目录下所有的内容，并到编译安装的目录执行 make clean 清理编译生成的文件。
