                                              # Nexus 的安装与启动（Linux）

                                              1. 下载 Nexus

                                                下载地址：https://help.sonatype.com/repomanager3/product-information/downloadopen in new window

                                                下载 Unix archive 。

                                              2. 解压到安装目录

                                              3. 修改环境变量（可选）

                                              4. 启动服务

                                                使用 nexus start 启动 Nexus。

                                              可以把 Nexus 作为系统服务启动，具体参考 https://help.sonatype.com/repomanager3/installation-and-upgrades/run-as-a-serviceopen in new window

                                              # Nexus 配置

                                              • 基础配置

                                                修改程序的 etc 目录中 nexus-default.properties 文件

                                              • 服务器启动配置

                                                修改程序的 bin 目录中 nexus.vmoptions 文件

                                              # Nexus 仓库

                                              # 仓库分类

                                              • hosted 宿主仓库

                                                可以保存无法从中央仓库获取的资源。

                                              • proxy 代理仓库

                                                代理远程仓库，通过 nexus 访问其他公共仓库。

                                              • group 仓库组

                                                将若干个仓库组成一个群组，简化配置。

                                                注意：仓库组不能保存资源，属于设计型仓库。

                                              # IDEA 中使用 Nexus

                                              在本地仓库配置访问私服的用户信息及下载地址。

                                              在工程配置上传地址。

                                              # 配置本地仓库

                                              • 访问私服的权限

                                                在 <settings> 标签的 <servers> 子标签内配置。

                                                <servers>
    <server>
        <id>maven-release</id>
        <username>admin</username>
        <password>admin</password>
    </server>
    <server>
        <id>maven-snapshot</id>
        <username>admin</username>
        <password>admin</password>
    </server>
</servers>

                                              • 下载地址

                                                在 <settings> 标签的 <mirrors> 子标签内配置。

                                                <mirrors>
    <mirror>
        <id>nexus-floli</id>
        <mirrorOf>*</mirrorOf>
        <name>Nexus Floli</name>
        <!--从私服复制-->
        <url>http://localhost/repository/maven-public/</url>
    </mirror>
</mirrors>

                                              # 配置工程

                                              在父工程添加标签 <distributionManagement>。

                                              <distributionManagement>
    <repository>
        <id>maven-release</id> <!--根据这个获取权限-->
        <url>http://localhost/repository/maven-release/</url>
    </repository>
    <snapshotRepository>
        <id>maven-snapshot</id>
        <url>http://localhost/repository/maven-snapshot/</url>
    </snapshotRepository>
</distributionManagement>
