                                                    # 选项卡

                                                    将选项卡包装在 tabs 容器中。

                                                    可以在 tabs 容器中添加一个 id 后缀，所有具有相同 id 的选项卡将共享相同的切换事件。

                                                    ::: tabs#fruit
<!-- fruit 是这个选项卡的 id，它是可选的 -->

<!-- 选项卡内容 -->
:::

                                                    在容器中使用 @tag 来标记和分隔选项卡内容。

                                                    在 @tag 标记后，可以添加 :active 默认激活选项卡，之后的文本将被解析为选项卡标题。

                                                    ::: tabs

@tab 标题 1

<!-- tab 1 内容 -->

@tab 标题 2

<!-- tab 2 内容 -->

@tab:active 标题 3

<!-- tab 3 将会被默认激活 -->

<!-- tab 3 内容 -->

:::

                                                    默认情况下，标题将用作选项卡的值，但可以使用 id 后缀覆盖它。

                                                    ::: tabs

@tab 标题 1

<!-- 此处，选项卡 1 的标题“标题 1”将用作值。 -->

<!-- tab 1 内容 -->

@tab 标题 2#值 2

<!-- 这里，tab 2 的标题将是 “标题 2”，但它会使用 “值 2” 作为选项卡的值-->

<!-- tab 2 内容 -->

:::

                                                    # 属性支持

                                                    使用语法 {attrs} 为 Markdown 元素添加属性。

                                                    • 想要一个 id 为 say-hello-world，文字为 Hello World 的二级标题

                                                      ## Hello World {#say-hello-world}

                                                    • 想要一个有 full-width Class 的图片

                                                      ![img](/image.png) {.full-width}

                                                    • 其他属性也收到支持

                                                      一个包含文字的段落。 {#p .a .b align=center customize-attr="content with spaces"}

                                                      渲染成：

                                                      <p id="p" class="a b" align="center" customize-attr="content with spaces">
  一个包含文字的段落。
</p>

                                                    # Badge

                                                    自定义颜色徽章。

                                                    使用 <Badge /> 添加徽章。

                                                    属性有：

                                                    属性解释必填类型备注
                                                    text徽章的文字是字符串类型
                                                    type徽章类型否-默认为 "info"
                                                    color徽章的颜色否字符串类型需要填入合法的 CSS 颜色
                                                    vertical徽章的方向否-默认为 "top"

                                                    属性 type 的可选类型有 "tip" | "warning" | "danger" | "info" | "note"

                                                    属性 vertical 的可选类型有 "top" | "middle"

                                                    徽章类型：

                                                    • tip
                                                    • warning
                                                    • danger
                                                    • info
                                                    • note

                                                    # 示例

                                                    Badge Test <Badge text="Building" type="warning" /> <Badge text="Ackerven" color="grey" />

                                                    Badge Test Building MrHope

                                                    # Catalog

                                                    使用 <AutoCatalog /> 添加自动生成目录。

                                                    属性有：

                                                    属性描述必填类型备注/默认值
                                                    base目录基础路径是字符串类型
                                                    levelCatalog 的最大层级否-默认值为 3
                                                    titleGetter标题获取器否(meta: RouteMeta) => string(meta: RouteMeta) => meta["title"]
                                                    iconGetter图标获取器否(meta: RouteMeta) => string(meta: RouteMeta) => meta["icon"]
                                                    orderGetter顺序获取器否(meta: RouteMeta) => string(meta: RouteMeta) => meta["order"]
                                                    shouldIndex是否应该被索引否(meta: RouteMeta) => boolean(meta: RouteMeta) => meta["index"] !== false

                                                    属性 level 的可选类型有 1 | 2 | 3

                                                    # 示例

                                                    <AutoCatalog base='/' />

                                                    # 脚注

                                                    使 Markdown 支持脚注。

                                                    用法：

                                                    • 使用 [^锚点文字] 定义脚注
                                                    • 在之后的任何位置使用 [^锚点文字]: ... 描述脚注
                                                    • 如果脚注包含多个段落，其后的段落应当保持双层缩进

                                                    # 示例

                                                    脚注 1 链接[1]。

                                                    脚注 2 链接[2]。

                                                    行内的脚注[3] 定义。

                                                    重复的页脚定义[2:1]。

                                                    脚注 1 链接[^first]。

脚注 2 链接[^second]。

行内的脚注^[行内脚注文本] 定义。

重复的页脚定义[^second]。

[^first]: 脚注 **可以包含特殊标记**

    也可以由多个段落组成

[^second]: 脚注文字。
